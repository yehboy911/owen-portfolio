Root Cause Analysis — MacBook Unibody A1342 Mass Production

Root Cause Analysis Factory QA Collaboration Risk Mitigation Process Standardization Quality Assurance
View Reference Page

Background

During Apple MacBook mass production, memory-related Fail Rate suddenly spiked from a stable 0.2% to over 25%, causing a shipping hold with over 100 failed units accumulated.

Pain Points

  • L10 system-level analysis showed random Fail/Pass even within the same part number
  • No visibility into DRAM lot code at system level
  • Traditional DOE/A/B testing failed to isolate root cause

Goal

Quickly identify the exact defective memory lot, unblock shipment, and establish a reusable analysis standard to prevent future quality escapes.

Solutions

As System Integration Testing Engineer, I partnered with factory QA and Test Engineers to redesign the analysis workflow from system level back to board level + lot traceability.

  • Shifted analysis from L10 system to L6 board level for full lot code visibility
  • Integrated L6 test results with ShopFlow MES data via PCB serial number
  • Redesigned A/B grouping from part number to lot code level with Excel visualization
  • Conducted cross-check and sample re-test with QA/Test for validation

Outcome & Impact

Quality & Stability

Isolated specific high-fail memory lot and restored Fail Rate from 25%+ back to 0.2% within weeks.

Risk Mitigation

Unblocked shipping hold, prevented potential recall risk, and resolved over 100 failed units in one go.

Engineering Efficiency

Created standardized PDCA "L6 + ShopFlow + lot-based analysis" process, now adopted as factory best practice for similar issues.

Cross-Functional Leadership

Led collaboration between factory QA, Test Engineers, and production lines to turn crisis into reusable methodology.
