Firmware Validation SoC Integration NPI Testing Cross-SoC Communication CLI Automation
View Product Page

Background

AWS first-generation Graviton ARM-based SoC platform with unique phased roadmap: qualify Annapurna-designed K2V4 Nitro card SoC on network card first, then migrate silicon to motherboard as main CPU.

Pain Points

  • Complex inter-SoC communication between Graviton main SoC and K2 Nitro card SoC
  • All operations executed via pure SSH CLI in Embedded OS (no UI)
  • High integration risk during silicon migration to full motherboard

Goal

Deliver full qualification of the new ARM SoC architecture and build a repeatable test framework to de-risk silicon migration and ensure cloud-grade stability.

Solutions

As Senior System Integration Engineer, I designed and executed the end-to-end testing strategy for this novel dual-SoC platform, collaborating closely with RD and AWS teams.

  • Developed comprehensive test plan covering functionality, stability, and performance on K2V4 Nitro card platform
  • Validated critical communication paths between Graviton SoC and Nitro card SoC using pure SSH CLI in Embedded OS
  • Established BMC-based monitoring framework to track system health and meet AWS qualification standards
  • Conducted firmware validation for device, sensor, and storage interactions under various workloads

Outcome & Impact

Quality & Stability

Successfully qualified first-generation Graviton SoC on Nitro card, achieving all functionality, stability, and performance requirements.

Risk Mitigation

Mitigated major integration risk by completing thorough card-level validation before silicon migration to motherboard.

Engineering Efficiency

Created reusable CLI-based testing methodology for Embedded OS environments, enabling efficient validation without GUI dependency.

Cross-Functional Leadership

Aligned RD, hardware, and AWS stakeholders; successful qualification led to our team being awarded continued responsibility for second-generation Graviton platforms.
