View Product Page

Background

New 1U server architecture integrating Mac mini controlled via Nitro Card — completely unfamiliar to both RD and the team.

Pain Points

  • No standard testing method or log collection process
  • Manual long-cycle testing extremely labor-intensive
  • Unclear data flow between Nitro Card and Mac mini

Goal

Build a repeatable, customer-approved test framework and automate long-duration validation to ensure stability while halving test resource cost.

Solutions

As Senior System Integration Engineer, I clarified the entire control path, designed the end-to-end test plan, and drove automation with RD and AWS stakeholders.

  • Clarified SSH-to-Nitro-to-Mac mini data flow and compiled critical bring-up & reliability checkpoints
  • Led creation of full server test plan covering functional, AC cycle, and thermal cycle scenarios; secured AWS approval after multiple iterations
  • Automated long-cycle logging using Nitro Card + Combin scripts, eliminating manual login and monitoring

Outcome & Impact

Quality & Stability

Delivered first repeatable test framework for the novel architecture, achieving zero validation gaps.

Engineering Efficiency

Reduced required test machines by 50%, halving validation cost and engineer manual effort.

Risk Mitigation

Eliminated long-term stability risk for AWS cloud deployment through automated, traceable logging.

Cross-Functional Leadership

Aligned RD, test team, and AWS customer expectations on a new platform.
